Local cheat sheet for your Neovim config

Keys, plugins, and how to use them

Leader key is Space. This page is based on your files in ~/.config/nvim, including the commented-out plugin imports in init.lua.

Quick Start

What Keys Use
Search files <leader>sf Find files with Telescope.
Live grep <leader>gl Search text across the project.
Buffers Tab / Shift-Tab Cycle through open buffers.
Close buffer <leader>x Delete the current buffer without closing Neovim.
Terminal <leader>t Open the floating terminal.
Git UI <leader>lg Open LazyGit.

Rules Of Thumb

j and k move by visual lines when wrapping is on.

jj or jk exits insert mode and saves.

<leader>lw toggles wrapped lines and horizontal scrolling mode.

<Esc> in normal mode clears search highlight.

Some mappings overlap by design or by accident. This guide reflects the actual current config, not idealized defaults.

Autocomplete, Editing, Treesitter

Keys Action How to use it
<C-Space> Manual completion Open completion suggestions from nvim-cmp.
<CR> Confirm completion Accept selected completion item.
Tab / Shift-Tab in insert mode Completion navigation Move through suggestions or jump through snippets.
<leader>ta Toggle autocomplete Turns automatic completion on or off persistently.
<C-/> Comment line/selection Toggle comments in normal or visual mode.
ys, cs, ds Surround Use default nvim-surround mappings to add, change, or delete surroundings.
gnn, grn, grc, grm Incremental selection Start and grow or shrink Treesitter-based selection.
aa, ia, af, if, ac, ic Textobjects Select arguments, functions, and classes.

Auto-working helpers: nvim-autopairs inserts matching brackets and quotes, nvim-ts-autotag closes and renames HTML/JSX tags, vim-sleuth detects indentation, todo-comments highlights TODO/FIXME/NOTE, and nvim-colorizer previews color codes inline.

Plugin Guide

These are the top-level plugins in your config. “Disabled” means the file exists but the import is commented out in init.lua.

alpha-nvim

Enabled

Startup dashboard.

  • Shows your custom header when Neovim opens without a file.
  • No special keymap configured here.

nvim-cmp + LuaSnip

Enabled

Completion and snippet engine.

  • Type in insert mode to see suggestions.
  • Use Tab, Shift-Tab, Enter, and <C-Space>.
  • <leader>ta toggles autocomplete state.

bufferline.nvim

Enabled

Buffer tabs across the top.

  • Tab and Shift-Tab switch buffers.
  • <leader>1..9 jumps to a specific buffer slot.

themery + themes

Enabled

Theme switcher.

  • <leader>ct opens the selector.
  • Preview and apply installed colorschemes.

Comment.nvim

Enabled

Comment helper.

  • <C-/> toggles the current line.
  • In visual mode it comments the selection.

Gitsigns

Enabled

Git change indicators in the gutter.

  • <leader>gp previews the hunk.
  • <leader>gt toggles blame.

indent-blankline

Enabled

Indent guides.

  • Works automatically in supported buffers.
  • Uses the ▏ character for indentation markers.

LazyGit

Enabled

Git TUI integration.

  • <leader>lg opens it.
  • Best for commits, staging, branches, and history.

lualine

Enabled

Statusline.

  • Shows mode, branch, diff, diagnostics, encoding, filetype, progress, and cursor location.
  • No extra keymaps configured.

nvim-lspconfig + Mason

Enabled

LSP server management and editor intelligence.

  • Servers are installed automatically through Mason tooling.
  • Hover, rename, definitions, references, symbols, and format-on-save come from here.

none-ls

Enabled

Formatter and linter bridge.

  • Formats on save through null-ls filtering.
  • Configured tools include Prettier, Stylua, Ruff, Go formatters, shfmt, and sqlfluff.

Telescope

Enabled

Fuzzy finder and picker UI.

  • Main entry points are <leader>sf, <leader>gl, and <leader>/.
  • Inside Telescope, use <C-j> and <C-k> to move.

tiny-inline-diagnostic

Enabled

Inline diagnostic rendering.

  • Displays diagnostics next to code instead of classic virtual text.
  • Use the diagnostic toggle keys to control it.

Undotree

Enabled

Undo history viewer.

  • <leader>u toggles the tree.
  • Your config resizes the panel after opening.

Yazi

Enabled

External file manager inside Neovim workflow.

  • \ opens Yazi at the current file.
  • This is your active file manager integration because Neo-tree is commented out.

render-markdown

Enabled

Markdown rendering polish.

  • Enhances Markdown display using Treesitter.
  • No explicit keymap in your config.

toggleterm.nvim

Enabled

Floating terminal.

  • <leader>t opens it.
  • Terminal closes with Esc or <leader>t while focused.

nvim-treesitter

Enabled

Syntax tree parsing.

  • Powers highlighting, textobjects, incremental selection, and structural motion.
  • Most of the syntax-aware keymaps come from here.

nvim-autopairs, ts-autotag, surround, sleuth, todo-comments, colorizer

Enabled

Editing helpers from extra-plugins.lua.

  • Most work automatically with little or no setup.
  • Surround is the main one with user-facing keymaps.

neo-tree.nvim

Disabled in init.lua

Alternative file explorer. Present in the repo but not currently loaded.

  • If enabled, <leader>w opens floating Neo-tree.
  • <leader>e opens the left sidebar explorer.
  • <leader>ngs opens the Git status view.
  • Inside the tree, keys like a, d, r, y, x, p, /, H handle file actions.

nvim-dap + dap-ui

Disabled in init.lua

Debugger integration. Present in the repo but not currently loaded.

  • F5 continue, F1 step into, F2 step over, F3 step out.
  • F7 toggles the debug UI.
  • <leader>b and <leader>B set breakpoints if enabled.
If you enable debug.lua, note that <leader>b already means “new buffer” in your core mappings, so you would introduce a mapping conflict.
